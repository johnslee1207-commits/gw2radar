# MetaBattle Benchmark

## Reference Links

- https://metabattle.com/wiki/MetaBattle_Wiki
- https://metabattle.com/wiki/Open_World

## Core Features

- Broad GW2 build database.
- Covers PvE, PvP, WvW, and Open World.
- Uses build quality bands such as meta, great, and good.
- Strong mode and role categorization.

## Data Model Signals

- `BuildSourceRegistry`: source type, allowed use, attribution, review requirement.
- `BuildQualityBand`: meta, great, good, beginner, open world, unknown.
- `ModeCoverage`: PvE, strike, raid, fractal, open world, PvP, WvW.

## Strengths

- Broad game-mode coverage.
- Useful for finding practical builds beyond high-end encounter metas.
- Good signal for returner-friendly and open-world options.

## Weaknesses For GW2Radar Positioning

- Community-curated data may vary in review depth.
- Build quality labels are source context, not personal account readiness.

## What GW2Radar Should Copy

- Mode coverage taxonomy.
- Build quality and difficulty hints.
- Source freshness and review requirements.

## What GW2Radar Should Avoid

- Over-trusting unreviewed community build data.
- Treating quality bands as absolute recommendations.
- Copying guide bodies instead of storing concise metadata and source links.

## Differentiation

GW2Radar should combine MetaBattle-like coverage with account-specific fit, budget alternatives, reviewed evidence, and freshness gates.
